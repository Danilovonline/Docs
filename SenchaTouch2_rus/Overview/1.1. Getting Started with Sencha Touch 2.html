Что такое Sencha Touch?

Sencha Touch позволяет быстро и легко создавать мобильные прилложения на основе HTML5, ко торые работают на Android, iOS и Blackberry устройствах, а также натиыне приложения для браузера.

Что вам потребуется

Для начала нужно скачать Sencha Touch 2 SDK и SDK Tools с сайта Сенчи.
Также потребуются:
  • Локальный WEB-сервер
  • Современный браузер; рекомендуются Chrome или Safari
Если в качестве web-сервера вы используете IIS под Windows, обратите внимание, что нужно дообавть Mime-тип application/x-json. Как это сделать найдете здесь: http://stackoverflow.com/a/1121114/273985

Установка

Для начала распокуйте архив с SDK в папку с вашим проектом. В идеале, эта папка должна быть доступна вашему HTTP-сервер. Например, через браузер вы должны зайти на адрес http://localhost/sencha-touch-2.0.0-gpl и увидеть документацию Сенчи
Также нужно запустить установщик SDK Tools. Установщик SDK Tools добавит поддержку коммандной строки Сенчи, что бы вы могли создавать новые приложения из шаблона. Проверьте как прошла установка, в коммандной строке перейдите в каталог Сенчи и выполните комманду сенчи. Например: 
        $ cd ~/webroot/sencha-touch-2.0.0-gpl/
        $ sencha
        Sencha Command v2.0.0 for Sencha Touch 2
        Copyright (c) 2012 Sencha Inc.
        ...
Примечание: Вы должны находитсявнутри каталога SDK или в каталоге сгенерированного проектаа.

Генерация вашего первого проекта

Ну что ж, у вас есть установленные Sencha Touch и SDK Tool, давайте сгенерируем приложение. Убедитесь, что вы по-прежнему в папке Сенча Touch SDK и введите следующее: 
        $ sencha generate app GS ../GS
        [INFO] Created file /Users/nickpoulden/Projects/sencha/GS/.senchasdk
        [INFO] Created file /Users/nickpoulden/Projects/sencha/GS/index.html
        [INFO] Created file /Users/nickpoulden/Projects/sencha/GS/app.js
        ...
Так сгенерируется скелет приложения с именным пространством GS(сокращенно от Getting Started) и директорию ../GS(на один уровень выше директории Sencha Touch SDK). Скилет приложения содержит все необходимые файлы Touch приложения, включая index.html, копию Touch SDK, CSS, картинок и конфигурационных файлов для упаковки вашего приложения в нативный код.
Дававайте проверим все ли успешно сгенерировалос, откроем приложение в браузере. Если предположить, что SDK был распаковон в корневую папку(webroot), вы должны открыть страницу http://localhost/GS и увидеть:

Изучим код

Откроем директорию GS в вашем любимом IDE или текстовом редакторе. Дерево директорий выглядит так:

Опишем каждый файл и деркторию:
  • app - директория содержит модели(Models), представления(Views), контроллеры(Controllers) и хранилища(Stores) вашего приложения
  • app.js - главный Javascript, "входные ворота" вашего приложения.
  • app.json - конфигурационный файл вашего приложения - используется конструктором для создания сжатой версии вашего приложения
  • index.html - HTML-файвашего приложения
  • packager.json - конфигурационный файл, используемый компоновщиком для создания навтивной версии вашего приложения для iOS и Android аппстора.
  • resources - директория содержит CSS и изображения для вашего приложения

        строка  8: пространство имен вашего приложения.
            Имена всех классов в вашем приложении должны начинаться с "GS", например, GS.view.Main

        строка 26: Функция, которая вызывается при инициализации(старте) вашего приложения. Здесь мы скрываем
            индикатор загрузки и отображаем представление "Main"
        строка 34: Эта функция вызывается когда есть обновления для вашего приложения и оно установленно у пользователя
Функция launch - это точка входа в ваше приложение. По умолчанию в приложении вы сначала скрываете индикатор загрузки, потом создаете экземпляр Main представления и отображаете его(добавлляете к видимому экрану(Viewport)).
Viewport это макет карточки к которой можно добавить компаненты. По умолчанию приложение отображает главное(Main) представление. Довайте посмотрим на код внутри.
